# Official Assessment Content Lifecycle

Date: 2026-07-18

## Lifecycle

Official Vytte content follows:

Draft → Internal review → Approved → Published immutable version → Archived or superseded by a new version.

Published content is never edited in place.

## Question versions

Question versions represent exact immutable wording and response structure.

A question version includes:

- question text;
- response type;
- answer options;
- numeric configuration and bands, when applicable;
- observation requirement;
- respondent role hint;
- methodology notes;
- source summary;
- review notes;
- reviewed by;
- approved by;
- effective date;
- content hash;
- publication timestamp.

Only approved question versions can be published.

Draft question versions cannot be placed into a published official framework.

## Framework versions

Framework versions may be:

- department frameworks for comprehensive facility assessment composition;
- focused frameworks for one health topic, programme, intervention, or scope.

Each framework version includes:

- purpose;
- sections;
- indicators;
- question placements;
- evidence expectations;
- applicability rules;
- scoring contribution;
- criticality;
- methodology notes;
- source summary;
- review notes;
- version;
- status;
- content hash.

Only exact published question versions may be used in a published framework version.

## Catalogue releases

Catalogue releases are published Vytte packages that pin exact framework versions.

Comprehensive Health Assessment uses catalogue releases to compose multiple department framework versions.

Focused Health Assessment uses a catalogue release that points to a deliberately designed focused framework. It is not created by blindly collecting every question attached to a topic.

## AI and methodology

AI may assist drafting and review preparation, but AI cannot:

- approve official questions;
- publish official content;
- define official scoring;
- claim compliance with a standard;
- invent sources;
- convert customer content into official Vytte methodology.

## Lightweight standards model

Vytte stores framework-level methodology notes and source summaries.

It does not require page-level provenance for every question in this phase.

Licensed or proprietary instruments must not be copied or redistributed without authorization.
